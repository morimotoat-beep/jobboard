# 研究者向け求人媒体サイト 構築指示書（v1）

## 1. 概要
研究者の就職・公募を支援する求人情報サイト。アカデミアノート（academianote.site）と連携する。

- 求人を出す側（研究室・大学）が**自分で自由に掲載**する。
- 探す側は**フィルターで検索**し、応募は各機関の**公式ページ（外部リンク）へ遷移**する。
- 運営者はほぼ手をかけず、求人は**締切やリンク切れで自動的に整理**される。
- **WordPressは使わない。**

**最重要の設計原則：求職者（応募者）の個人情報は一切収集しない。**
応募導線は外部サイトへ丸投げする。これは職業安定法上の「特定募集情報等提供事業（＝厚労省への届出義務あり）」に該当しないようにするため。会員登録・お気に入り保存・求職者向けメルマガなどは実装しない。

---

## 2. 技術スタック
- フロント／サーバー：**Next.js（App Router, TypeScript）**
- DB・認証・メール送信：**Supabase（PostgreSQL ＋ Magic Link 認証）**。メール送信は Resend 等でも可。
- ホスティング・定期実行：**Vercel（本体ホスティング ＋ Vercel Cron）**
- 多言語：**next-intl**（または next-i18next）。日・英・中・韓の4言語。
- ドメイン：サブドメイン **`jobs.academianote.site`** を想定。

---

## 3. ページ構成（各言語版あり）
- トップ／求人一覧（フィルター付き検索）
- 求人詳細（要約 ＋「応募はこちら」で外部URLへ）
- 求人投稿フォーム（求人側のみメール認証）
- 投稿者用の編集・削除ページ（メールのトークンリンクから開く）
- 利用規約・プライバシーポリシー
- 管理画面（運営者のみ・最小限。通報対応用）

---

## 4. データモデル（listings テーブル）
| カラム | 内容 |
|---|---|
| id | 主キー |
| title | タイトル（自由文・投稿言語のまま） |
| summary | 要約（自由文・投稿言語のまま） |
| field | 研究分野コード（§9参照） |
| job_type | 職種コード（§9参照） |
| employment_type | 雇用形態コード（§9参照） |
| country | 勤務地・国 |
| prefecture | 都道府県（日本国内の場合のみ・任意） |
| deadline | 締切日 |
| external_url | 応募・詳細の外部リンク |
| post_language | 投稿言語（ja/en/zh/ko・言語バッジ表示用） |
| poster_email | 投稿者の連絡先・認証用（求人側のみ・一般非公開） |
| status | draft / published / expired / link_flagged / hidden |
| edit_token | 編集・削除用トークン |
| report_count | 通報数 |
| created_at / updated_at | 作成・更新日時 |

※**求職者情報を保存するテーブルは作らない。**

---

## 5. 投稿フロー（メール認証つき）
1. 求人側が投稿フォームを入力（§4の項目）。
2. 確認メール（Magic Link）を送信。リンクを押すと `status=published` で公開。
3. 同じメールに**編集・削除用リンク（edit_token付き）**を記載し、投稿者が自分で管理できるようにする。
4. スパム対策：メール認証に加え、フォームに**ハニーポット**＋**投稿レート制限**を入れる。

---

## 6. 検索・フィルター
すべて**選択式（ドロップダウン／タグ）**。自由入力は使わない（＝言語をまたいだ検索を可能にするため）。

- 研究分野（§9）
- 職種（§9）
- 雇用形態（§9）
- 勤務地（国 → 日本の場合は都道府県）
- 締切（〜まで／締切間近 など）
- キーワード（任意・タイトル/要約に対して）

**公開ルール：`status=published` かつ締切前のものだけ一覧に表示。** expired / hidden / link_flagged は非表示。

---

## 7. 自動メンテナンス（Vercel Cron／1日1回）
- **締切超過**：`deadline` を過ぎた求人を `status=expired` に変更（非表示・データは保持）。
- **リンク切れチェック**：`external_url` にHTTPアクセスし、404/410等が**3日連続**したら `status=link_flagged` にし、投稿者へ自動通知メール。誤検知に備え**即削除はしない**。
- 注意：大学サイトはボットを弾く／募集終了後もページを残すことがあるため、**締切日を主たる削除トリガー**にする。リンク切れは補助的な扱い。

---

## 8. 多言語方針
- **UI（ボタン・メニュー・フィルター名）**：4言語フル翻訳。
- **フィルター項目**：内部コード ＋ 各言語ラベル（§9）。これにより、何語で投稿された求人でも全言語のユーザーから検索できる。
- **自由文（タイトル・要約）**：投稿言語のまま表示し、「日本語／English」等の**言語バッジ**を付ける。自動翻訳はv1では実装しない（Phase2候補）。
- 言語はv1で **ja / en / zh / ko**。スペイン語は後から追加できる構成にしておく。

## 10. 通報・管理（安全装置）
- 各求人に小さく**「通報」ボタン**を置く。押すと `report_count` を加算（求職者情報は取らない）。
- 一定数を超える or 管理画面で運営者が確認 → 不適切なら `status=hidden` にできる。
- 管理画面は運営者のみ（Supabase Auth で限定）。普段は操作不要の“安全装置”。

---

## 11. 法令・プライバシー
- **求職者の個人情報を収集しない設計を厳守**（届出義務の回避）。
- 求人情報は正確に表示する（募集情報等提供事業者としての一般ルール）。
- 利用規約に、掲載は投稿者の責任である旨・運営者の免責・運営者の削除権限を明記する。
- ※最終的な法令確認は専門家（社労士等）で行う。

---

## 12. セキュリティ
- Supabase／メールAPIのキーは `.env` ／ Vercel環境変数に置き、リポジトリにコミットしない。
- Supabase の **service role key はサーバー側のみ**。クライアントには anon key のみ。
- 投稿・編集はトークンまたは認証必須。管理画面は認証ゲートを通す。
